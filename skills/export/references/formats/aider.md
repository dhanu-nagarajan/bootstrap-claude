# Aider Export Format

## Output Structure

Single file: `.aider.conventions.md` at project root.

## Context Budget

Aider includes this file in every prompt alongside its own repo-map. The repo-map already provides structural awareness (file list, function signatures, class hierarchies), so focus on what Aider cannot infer automatically:

- Naming conventions and style preferences
- Architecture boundaries and dependency direction rules
- Forbidden patterns and anti-patterns
- Testing philosophy and patterns
- Error handling conventions

Do NOT repeat information Aider already gets from the repo-map (file structure, function signatures, import graphs).

## File Structure

```markdown
# Project Conventions

## Architecture Rules
[Module boundaries, dependency direction, layer separation]

## Coding Conventions
[Naming, typing, import ordering, code style]

## Error Handling
[How errors are created, propagated, and handled]

## Testing
[Framework, naming, patterns, what to test]

## Forbidden
[Flat list of anti-patterns]
```

## Content Guidelines

- **Target 100-200 lines.** Aider prepends this to every prompt — keep it tight.
- **No structural overview.** Aider's repo-map handles this. Don't describe what files exist or what modules do.
- **Directive tone.** Every line should be an instruction: "Use X", "Never Y", "Always Z."
- **Group by concern.** Developers skim this file; clear headings let them find rules fast.
- **Skip workflow.** Aider handles its own edit/test cycle. Don't include commit conventions or PR processes.
- **No examples longer than 3 lines.** If a pattern needs a long example, describe it in words instead.

## Forbidden Section

Always include. One line per rule, prefixed with "Never":

```markdown
## Forbidden

- Never use `any` type — use `unknown` and narrow
- Never mutate function arguments
- Never use synchronous file I/O in request handlers
```

## Handling Existing File

If `.aider.conventions.md` already exists:
- Read it first
- Preserve any sections not covered by the export
- Add a comment at the top: `<!-- Generated by bootstrap-claude /export -->`
