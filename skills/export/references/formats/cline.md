# Cline Export Format

## Output Structure

Single file: `.clinerules` at project root.

## Context Budget

Cline is an agentic coding tool similar to Claude Code. It reads `.clinerules` on every task and supports multi-step workflows. Include both code standards AND workflow guidance.

## File Structure

```markdown
# Project Rules

## Overview
[1-3 sentences: what this project is, primary language, framework]

## Architecture
[Module structure, dependency rules, key boundaries]
[Where to put new files, how modules communicate]

## Coding Standards
[Naming, typing, imports, error handling]
[Language-specific conventions]

## Forbidden Patterns
[Flat list of hard rejections]

## Testing
[Framework, naming, what to test, how to run]

## Pre-Action Checklist
[Steps to verify before making changes — adapted from pre-commit.md]

## Common Tasks
[Patterns for frequent operations: adding an endpoint, creating a component, etc.]
```

## Content Guidelines

- **Target 200-400 lines.** Cline has a reasonable context budget and benefits from detail.
- **Include workflow guidance.** Unlike Copilot, Cline performs multi-step tasks. Include pre-action checks and common task patterns.
- **Architecture section is critical.** Cline creates and modifies files — it needs to know WHERE things go.
- **Include actual commands.** Build, test, lint, and dev server commands from the project.
- **Directive tone.** "Do X" not "You might want to consider X."

## Pre-Action Checklist

Adapt the `.claude/checklists/pre-commit.md` into a simplified checklist:

```markdown
## Pre-Action Checklist

Before submitting changes, verify:
- [ ] Types pass: `npm run typecheck` (or equivalent)
- [ ] Tests pass: `npm test` (or equivalent)
- [ ] No lint errors: `npm run lint` (or equivalent)
- [ ] No forbidden patterns introduced
- [ ] New files are in the correct module directory
```

Use the project's actual commands.

## Common Tasks Section

Include 2-3 common task patterns based on project type:

```markdown
## Common Tasks

### Adding an API Endpoint
1. Create handler in `src/handlers/`
2. Add route in `src/routes.ts`
3. Add request/response types in `src/types/`
4. Write tests in `src/handlers/__tests__/`
```

Adapt to the project's actual structure.

## Handling Existing File

If `.clinerules` already exists:
- Read it first
- Preserve user-written content
- Add a comment at the top: `<!-- Generated by bootstrap-claude /export -->`
