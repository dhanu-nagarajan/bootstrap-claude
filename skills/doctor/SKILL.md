# Doctor: Health Check and Drift Detection

Detect drift between `.claude/` configuration files and the current codebase state. Identify broken references, coverage gaps, and stale content that reduces the effectiveness of the engineering system.

**The problem this solves:** Codebases evolve continuously — files get renamed, dependencies change, new modules appear, build commands update. The `.claude/` files generated by /bootstrap become stale over time, referencing things that no longer exist and missing things that do. This skill diagnoses exactly what drifted and what needs updating.

**Quality bar:** Every reported issue must be verifiable. Broken references must cite the `.claude/` file containing the reference and confirm the target does not exist. Coverage gaps must name the specific new element that is uncovered.

## Execution

### Step 1: Prerequisites Check

1. Verify `.claude/` directory exists
2. If missing, tell the user: "No `.claude/` directory found. Run /bootstrap first to generate it."
3. List all files in `.claude/` recursively — these define the health check scope

### Step 2: Reference Validation

Read the full doctor specification:

```
Read file: ${CLAUDE_PLUGIN_ROOT}/skills/doctor/references/doctor-spec.md
```

For every `.md` file in `.claude/`:

1. **Extract file path references** — Find all paths in backticks that look like file paths (contain `/` or `.` with an extension). Use `Glob` to check each exists. Missing files are broken references.
2. **Extract command references** — Find all shell commands in backticks or code blocks (starting with `npm`, `yarn`, `pnpm`, `make`, `cargo`, `go`, `python`, `pytest`, `docker`, etc.). Cross-reference with `package.json` scripts, `Makefile` targets, or equivalent. Missing commands are broken references.
3. **Extract code references** — Find all function names, type names, class names in backticks. Use `Grep` to verify each exists in the source code. Missing names are broken references.

Classify each broken reference:
- **Missing file** — Referenced path does not exist anywhere
- **Moved file** — Path does not exist but a file with the same name exists elsewhere (use `Glob` with just the filename)
- **Missing command** — Referenced command/script does not exist in package.json or Makefile
- **Missing code symbol** — Referenced function/type/class not found in source

### Step 3: Coverage Analysis

Check for codebase elements not covered by `.claude/` files:

1. **New directories** — List all top-level directories under `src/` (or equivalent). Check if each is mentioned in `architecture.md`. Unreferenced directories are coverage gaps.
2. **New dependencies** — Read the package manifest. Compare listed dependencies against mentions in `.claude/standards/` files. New dependencies not mentioned anywhere are gaps.
3. **New file patterns** — Use `Glob` to find file extensions and naming patterns. Check if each pattern is covered by `testing.md` or `language.md`. New patterns (e.g., `.graphql` files, `.proto` files) not mentioned are gaps.
4. **Changed commands** — Read `package.json` scripts (or equivalent). Compare with commands referenced in `.claude/checklists/`. Renamed or removed scripts are gaps.
5. **New config files** — Check for config files (`.env.example`, `docker-compose.yml`, CI configs) not mentioned in any `.claude/` file.

### Step 4: State Check

If `.claude/.bootstrap-state.json` exists:

1. Read the stored state
2. Compare `last_bootstrap` or `last_update` date with current date — report staleness
3. Compare `files_generated` list with current `.claude/` file list — detect manually added or removed files
4. Compare `analysis_fingerprint` with a fresh fingerprint of current codebase state — detect significant codebase changes since last bootstrap

If state file does not exist, note that no baseline exists for comparison.

### Step 5: Generate Report

Write `.claude/doctor-report.md`:

```
# Doctor Report -- [YYYY-MM-DD]

## Health Summary
- Files checked: [count of .claude/ files analyzed]
- Broken references: [count]
- Coverage gaps: [count]
- Stale content indicators: [count]
- Overall health: [HEALTHY | NEEDS UPDATE | STALE]

## Broken References

### [.claude/file-path.md]
| Line | Reference | Type | Status |
|---|---|---|---|
| 15 | `src/old/path.ts` | file path | MISSING (moved to `src/new/path.ts`) |
| 42 | `npm run old-script` | command | MISSING (not in package.json) |

## Coverage Gaps

### New Directories
- `src/webhooks/` -- not mentioned in architecture.md

### New Dependencies
- `zod` (v3.22) -- not mentioned in any standards file

### Changed Commands
- `npm run test:e2e` -- referenced in checklists but not in package.json

## Stale Content
- `.claude/standards/architecture.md` references 5 modules, codebase now has 8
- `.claude/.bootstrap-state.json` last updated 45 days ago

## Recommendations
| Priority | Action | Files Affected |
|---|---|---|
| HIGH | Run /update to fix broken references | [list] |
| MEDIUM | Add new modules to architecture.md | architecture.md |
| LOW | Review new dependencies for standards | language.md |
```

### Step 6: Summary

Present to the user:
1. Overall health status with a one-line assessment
2. Count of broken references, coverage gaps, and stale content
3. Top 3 most important issues to address
4. Recommended action: /update for auto-fixable issues, manual review for structural changes

## Quality Principles

1. **Verifiable** — Every reported issue can be independently confirmed
2. **No false alarms** — Do not report working references as broken
3. **Specific** — Name exact files, lines, and references — not "some files are stale"
4. **Actionable** — Every finding includes what to do about it
5. **Prioritized** — Broken references before coverage gaps before staleness
6. **Non-destructive** — Doctor only reports, never modifies files

**The test:** Could a developer look at the report and know exactly which files need updating and why?
